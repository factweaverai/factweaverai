# Contributing to FactWeaver Website

Thank you for your interest in contributing to the **FactWeaver** marketing website! We welcome contributions from the community to help us improve our documentation, design, and code.

## 🚀 Getting Started

To get started with the codebase, follow these steps:

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/YOUR_USERNAME/factweave.git
    cd factweave
    ```
3.  **Install dependencies**:
    ```bash
    npm install
    ```
4.  **Create a new branch** for your feature or fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```

## 🛠️ Development Workflow

1.  **Run the development server**:
    ```bash
    npm run dev
    ```
    Open [http://localhost:3000](http://localhost:3000) to view your changes.

2.  **Code Style**:
    -   We use **ESLint** and **Prettier** to maintain code quality.
    -   Please ensure your code passes linting before submitting:
        ```bash
        npm run lint
        ```

3.  **Commit your changes**:
    -   Write clear, concise commit messages.
    -   Example: `feat: Add new pricing section` or `fix: Correct typo in hero component`.

## 📥 Submitting a Pull Request

1.  Push your changes to your fork:
    ```bash
    git push origin feature/my-new-feature
    ```
2.  Open a **Pull Request (PR)** on the main repository.
3.  Provide a clear description of your changes, including screenshots if you modified the UI.
4.  Wait for review! We will do our best to review your PR as soon as possible.

## 🐛 Reporting Issues

If you find a bug or have a suggestion, please [open an issue](https://github.com/codegraftai/factweave/issues). Be sure to include:

-   A clear title and description.
-   Steps to reproduce the issue.
-   Browser/OS information if relevant.
