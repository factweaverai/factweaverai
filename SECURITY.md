# Security Policy

## Supported Versions

Since this is a web application (landing page), we only support the latest deployed version.

| Version | Supported |
| ------- | ------------------ |
| Latest | :white_check_mark: |
| Older | :x: |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in the FactWeaver website, please follow these steps:

1.  **Do NOT create a public GitHub issue.** This could expose the vulnerability before we have a chance to fix it.
2.  **Email our security team** directly at **security@factweaver.com** (or replace with appropriate contact).
3.  Include a detailed description of the vulnerability and steps to reproduce it.

## Our Process

1.  **Acknowledgement:** We will acknowledge your report within 48 hours.
2.  **Investigation:** We will investigate the issue and determine its impact.
3.  **Fix:** verified security issues will be prioritized and fixed in a timely manner.
4.  **Disclosure:** Once the vulnerability is patched, we will publicly disclose the issue if appropriate.

## Scope

This policy applies to the **FactWeaver marketing website** source code hosted in this repository. 

**Note:** If you find a vulnerability in the **FactWeaver SaaS Application/Engine** (the product itself), please refer to the security policy specific to that platform (usually found in the application footer or dedicated security page).

Thank you for helping keep FactWeaver secure!
